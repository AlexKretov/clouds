# Классификация облачных паттернов по спутниковым снимкам

Этот репозиторий содержит решение задачи классификации четырёх типов облачных образований — **Fish**, **Flower**, **Gravel** и **Sugar** — по спутниковым изображениям.

Ноутбук ([`clouds_nn.ipynb`](clouds_nn.ipynb)) реализует пайплайн глубокого обучения с использованием предобученной модели **ResNet34**, с акцентом на эффективную загрузку данных и оптимизацию производительности.

## 🎯 Постановка задачи
Задача представляет собой многоклассовую классификацию с множеством меток (**multi-label**). Каждое спутниковое изображение может содержать один или несколько облачных паттернов, и модель должна предсказать наличие каждого из четырёх классов. 

**Метрика оценки:** коэффициент Dice, усреднённый по всем классам.

## 📊 Набор данных
Набор данных взят из соревнования *Understanding Clouds from Satellite Images* на Kaggle.

*   **Обучающие изображения:** более 5 500 RGB-снимков размером 1400×2100 пикселей (для обучения уменьшены до 224×224).
*   **Метки:** каждому изображению соответствуют четыре бинарные метки (по одной на класс), полученные из RLE-масок.
*   **Обработка:** ноутбук парсит CSV-файл и формирует для каждого изображения multi-hot вектор меток.

## 🛠 Подход

### 1. Предобработка данных
*   **Ресайз:** Изображения приводятся к размеру 224×224.
*   **Кэширование:** Все обучающие и валидационные изображения после ресайза кэшируются в оперативной памяти в формате PIL, что устраняет узкое место ввода-вывода (I/O).
*   **Аугментация:**
    *   Случайное горизонтальное отражение.
    *   Случайный поворот (±10°).
    *   Смещение цвета (яркость, контраст, насыщенность, оттенок).
    *   Нормализация (ImageNet stats).

### 2. Архитектура модели
*   **Base:** ResNet34 (pretrained on ImageNet).
*   **Head:** Последний полносвязный слой заменён на выход размерности 4.
*   **Оптимизация:** Используется `torch.compile` для ускорения вычислений (опционально).

### 3. Обучение и оптимизация
*   **Loss:** `BCEWithLogitsLoss` (стандарт для multi-label).
*   **Оптимизатор:** `AdamW` (LR: 1e-3, Weight Decay: 1e-4).
*   **Scheduler:** `ReduceLROnPlateau` (фактор 0.5, терпение 3 эпохи).
*   **AMP:** Смешанная точность для ускорения и экономии VRAM.
*   **Early Stopping:** Остановка после 7 эпох без улучшения Dice.

## 🏆 Результаты
В ходе обучения (на 20% от общего объёма данных) лучший коэффициент Dice на валидационной выборке составил **~0.7483**. 

Модель сохранена в файл `best_model_classification.pth`. Визуализация предсказаний на 9 случайных примерах демонстрирует уверенное выделение паттернов с редкими ошибками на отсутствующих классах.

## 🚀 Запуск

1.  **Клонируйте репозиторий:**
    ```bash
    git clone https://github.com/yourusername/cloud-classification.git
    cd cloud-classification
    ```

2.  **Подготовьте данные:**
    Разместите датасет в папке `data/` со следующей структурой:
    ```text
    data/
    ├── train_images/          # Обучающие изображения
    └── train.csv              # Метки с RLE-масками
    ```

3.  **Запустите обучение:**
    Через Jupyter Notebook:
    ```bash
    jupyter notebook clouds_nn.ipynb
    ```
    Или как Python-скрипт:
    ```bash
    python clouds_nn.py
    ```

## ✨ Ключевые особенности
*   **In-memory caching:** Быстрый доступ к данным на каждой эпохе.
*   **Mixed Precision (AMP):** Снижает потребление видеопамяти.
*   **Early Stopping:** Предотвращает переобучение.
*   **Configurability:** Параметры (batch size, LR, epochs) легко меняются в блоке конфигурации.
